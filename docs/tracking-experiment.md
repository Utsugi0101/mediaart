# 物体認識の実験計画

更新日: 2026-07-17

## 研究質問

固定したiPad、朽木背景、固定照明という展示条件で撮影した専用データにより、用意した餌と障害物だけを、作品の反応に十分な精度と遅延で認識・追跡できるか。

## 仮説

最終設置に近い画像でインスタンスセグメンテーションモデルを微調整すれば、未調整の汎用モデルよりも、空の画面・手による遮蔽・物体の重なりを含む場面で誤検出を抑えつつ、中心位置誤差1%以内、E2E遅延150ms以内を達成できる。

## 比較対象

- ベースライン: 追加学習していない同系統の事前学習済みセグメンテーションモデル
- 提案方式: 展示専用データで微調整したインスタンスセグメンテーションモデル
- 追跡なし条件: フレームごとの検出結果をそのまま使用
- 追跡あり条件: 平滑化、デッドゾーン、追加確認、0.5秒の欠落猶予を使用

モデル系列や入力解像度を変える場合は、同じテスト集合で比較する。

## データセット

実物決定後に`data/raw/<dataset-version>/`へ撮影する。生成物と個人環境のデータはGitへ追加しない。

必須場面:

- 物体が0個の空画面
- 餌だけ、障害物だけ、両方がある配置
- 1個から多数まで、個数の異なる配置
- 画面中央、四隅、辺に接した配置
- 物体同士が近い・一部重なる配置
- 手で移動中、手で一時的に覆う場面
- 登録外の自然物や日用品だけを置いた負例
- シミュレーションの明暗・模様が物体の周囲で変化する場面
- 開場時から終了時まで想定した照明差と機器温度差

静止画だけでなく、移動、追加、取り除き、遮蔽を含む連続動画を撮影する。動画の近接フレームが学習側とテスト側へ漏れないよう、撮影セッション単位で分割する。

## アノテーションと分割

- クラスは`food`、`obstacle`。必要な場合だけ`hand`を補助クラスとして追加する
- 物体ごとに可視領域のマスクを付ける
- 画面外や判定困難な物体の扱いをデータセット内で統一する
- セッション単位でtrain / validation / testへ分割する
- testは最後まで固定し、閾値調整には使わない
- データセット版、撮影条件、分割一覧をメタデータへ記録する

## 評価指標

- クラス別mask precision / recall / mAP
- 空画面・手だけ・登録外物体での誤追加数
- 画面短辺に対する中心位置誤差
- 物体移動中のID switch数
- 一時遮蔽後に同じIDへ戻れる割合
- カメラ取得からWebSocket受信までの遅延
- M1 Mac miniでの推論更新レートとCPU/GPU/メモリ使用量
- 30分、最終的には10時間運転時のクラッシュ、メモリ増加、追跡ずれ

## 初期合格基準

- 登録物体の中心位置誤差: 画面短辺の1%以内
- 操作からシミュレーション反映まで: 150ms以内
- 推論・追跡結果の更新: M1 Mac miniで平均10Hz以上
- 空画面および手だけの評価動画: 誤って確定追加される物体が0件
- 0.5秒未満の遮蔽: ブラウザ上の物体を削除しない
- 30分連続試験: クラッシュなし、認識位置の累積ずれなし

精度指標の具体的なmAP閾値は、実物とアノテーションが確定した最初のベースライン測定後に固定する。作品体験に直結する誤追加、位置誤差、遅延を優先する。

## 固定条件と変数

実験中に固定するもの:

- iPadと画面の位置・角度
- 背景素材と保護板
- 照明位置、明るさ、露出、ホワイトバランス、フォーカス
- train / validation / test分割
- 評価動画と評価コード

比較するもの:

- モデルサイズ
- 入力解像度
- 信頼度閾値
- 平滑化係数、デッドゾーン、追加確認フレーム数、欠落猶予

## 再現性

- 乱数seedは`42`を既定とし、重要な比較は`42`、`43`、`44`で繰り返す
- 設定値はコマンドへ埋め込まず、版管理する設定ファイルへ置く
- 実行ごとにGit commit、設定、データセット版、モデル、seed、環境情報、指標を保存する
- 学習済み重み、データ、動画、実験ログは`data/`、`models/`、`runs/`へ置き、Gitから除外する
- 採用モデルと閾値は、比較表と理由を`runs`内のレポートへ残す
