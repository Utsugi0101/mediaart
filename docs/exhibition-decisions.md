# 展示構成と決定事項

更新日: 2026-07-17

## 作品と体験

- 作品名は「モジホコリ」
- 筑波大学「メディアアート」の展示作品として制作する
- 実物の餌・障害物を動かし、画面内の生命体の適応を観察する
- 作品は無音とする
- 説明は画面外の札へ置く
- 本番画面には文字、ボタン、カーソル、デバッグ表示を一切置かない

## 画面と造作

- ディスプレイは古いDell製27インチ液晶を予定する
- ディスプレイを水平にし、箱へ収める
- 背景表現は「朽ち木」で固定する
- 周囲は乾燥した土、朽木、落ち葉などを使い、取り外せる浅いトレイとして作る
- 濡れた土や水やりは行わない
- 画面の上に透明で交換・清掃可能な保護板を置く
- 保護板はディスプレイ本体ではなく箱の枠で支持し、画面とわずかな隙間を設ける
- 放熱空間を少なくとも約50mm確保し、実機の排気位置を塞がない
- 鑑賞者は正面・左・右の3方向から接近できる。背面は配線と支柱を置く機材側とする

ディスプレイ型番が判明したら、スタンドの外し方、VESA穴、外形、解像度、端子、排気位置を確認して箱の寸法を確定する。

## カメラと照明

- iPad Air（USB-C搭載、正確な世代は未確認）の背面カメラを使用する
- 画面中央の真上、保護板から約70cmを初期設置高さとする
- 背面の支柱へタブレットクランプで固定し、落下防止ワイヤーを併用する
- USB-C有線接続とCamo Camera / Camo StudioでMacへ入力する
- 左右2方向から拡散した照明を固定する
- 会場は暗めとし、カメラの露出、ホワイトバランス、フォーカスを固定する
- Camoの仮想カメラをOpenCVが取得できない場合はOBS Virtual Cameraを中継する

SIGMA fpと45mmレンズは、重量と必要な設置距離を考慮して本番候補から外す。

## コンピューターと運用

- 本番はM1 Mac miniを使用する
- 物体追跡とp5.jsシミュレーションを同じMacで動かす
- 学習・開発・予備にはMacBook Airを使用できる
- シミュレーションは30fps、認識結果の更新は約10Hzを目標とする
- 1日約10時間運転する
- 展示中は状態を引き継ぎ、一定間隔で保存する
- クラッシュ後は同日の直近状態から復元する
- 毎朝は新しい状態から開始する
- rawカメラ映像は展示運用中に保存しない
- 日々の記録として作品画面のスクリーンショットを残すことは許容する

## 物体認識

- 見えるマーカーは使わない
- RGB映像の専用インスタンスセグメンテーションを採用する
- こちらで事前に用意した物体だけを、最終環境に近い画像で学習する
- 物体の役割は`food`または`obstacle`へ固定登録する
- 実物は椎茸を含む可能性があるが、まだ確定していない
- 物体が変わる場合、事前の再撮影・再学習を許容する
- 同時物体数は0個以上の可変数とし、コードに上限・下限を設けない
- 未登録物体は無視する
- 手は遮蔽物として扱い、シミュレーション内の物体にはしない
- RGB方式で必要精度を得られない場合だけ深度カメラを再検討する

## シミュレーション

- 背景は「朽ち木」で固定する
- 成長速度は現状の20倍設定を採用する
- 物体を動かし終えた際に餌を補充する現状挙動は一旦維持する
- 餌がない場合、生命体は消滅せず、徐々に縮小・減速して休眠する
- 餌が戻ると活動を回復する
- カメラが停止した場合、最後に確定した物体配置を維持して動き続ける

## 安全と日次確認

- 保護板の傷、汚れ、たわみを清掃前後に確認する
- iPad、照明、支柱の固定と落下防止を毎朝確認する
- 排気口、電源タップ、USB-Cケーブルを土や造作で覆わない
- Macとディスプレイの温度上昇、映像復帰、追跡状態を開場前に確認する
- 運営用の状態確認は本番画面とは別の端末またはURLで行う
