# モジホコリ — 展示概要

## 概要

「モジホコリ」は、筑波大学の授業「メディアアート」のために制作している、変形菌（Physarum）に着想を得たインタラクティブ作品です。

水平なディスプレイの上を黄色い生命体が移動し、餌を探しながら自らの経路を網目状に組み替えます。鑑賞者がディスプレイ上の実物オブジェクトを動かすと、その配置をカメラが認識し、生命体の誘引方向や障害物が変化します。正解を目指す操作ではなく、環境に手を加え、少し待ち、予測しきれない応答を観察する作品です。

## 現在のプロトタイプ

ブラウザ上のp5.jsシミュレーションとして実装しています。現在は画面上の餌と障害物をマウスで動かせます。展示版では、この入力を上方カメラで検出した実物オブジェクトへ置き換えます。

追跡方式は、見えるマーカーを使わず、展示で実際に使う物体だけを学習したインスタンスセグメンテーションモデルを採用します。物体の種類は固定の役割である`food`または`obstacle`として登録しますが、置かれる個数は0個を含む可変数です。未登録の物や鑑賞者の手は、シミュレーション内の物体として扱いません。

## 最終展示のイメージ

- 27インチのディスプレイを箱の中へ水平に設置する
- 画面の上に、交換可能な透明保護板を置く
- 箱の周囲を乾燥した土、朽木、落ち葉などで構成する
- 約70cm上方にiPad Airを固定し、画面全体を撮影する
- iPadの映像をUSB-CとCamo StudioでM1 Mac miniへ入力する
- 左右から拡散した固定照明を当て、露出とホワイトバランスを固定する
- M1 Mac mini上で物体追跡とシミュレーションを同時に動かす

本番画面には文字、操作UI、カーソル、認識枠、仮想の餌・障害物を一切表示しません。説明は画面外の札に置き、運営用情報は別の画面またはURLで確認します。

## 展示中の振る舞い

- 背景は「朽ち木」、成長速度は現在の20倍設定で固定する
- 1日の展示中は状態を引き継ぎ、クラッシュ時も直近の保存状態から復旧する
- 毎朝新しい状態から開始し、約10時間連続稼働する
- 餌が0個でも生命体は消えず、徐々に縮小・減速して休眠する
- 餌が再び置かれると活動を回復する
- カメラ停止時は最後に確定した物体位置を保持し、シミュレーションを止めない
- 音は使用しない

## 開発方針

画像認識とp5.jsシミュレーションは別プロセスにします。追跡プロセスは物体一覧をWebSocketで送り、ブラウザは同じ物体IDの栄養量などを維持しながら座標を更新します。カメラや学習済みモデルがなくても開発できるよう、マウス入力とモック追跡入力も残します。

学習対象の実物はまだ確定していません。椎茸を含む可能性はありますが、一般的な「キノコ」や「石」を認識するのではなく、最終的に用意した個体と展示条件で撮影したデータから専用モデルを作ります。物体を変更する場合は事前の再撮影・再学習を許容します。
